# sshman — SSH 会话管理 CLI 工具设计文档

**日期:** 2026-06-12
**状态:** 已确认

---

## 概述

`sshman` 是一个 Python 3 CLI 工具，用于管理多个 SSH 会话。支持会话配置持久化（AES-256 加密 YAML）、自动登录（pexpect）、标签分类、批量操作、跳板机、端口转发、配置导入、操作日志、健康检查等功能。

## 二、会话数据模型

加密存储于 `~/.sshman/config.enc`（AES-256-GCM 密文），解密后为 YAML，结构如下：

```yaml
sessions:
  - name: prod-web-01          # 唯一标识
    host: 10.0.1.100
    port: 22
    user: admin
    password: ""               # 留空使用密钥或交互输入
    identity_file: ~/.ssh/id_ed25519
    tags: [prod, web, beijing]
    jumphost: bastion          # 引用另一 session name
    tunnels:
      - type: local            # -L
        local_port: 5432
        remote_host: 127.0.0.1
        remote_port: 5432
    notes: "生产 Web 服务器 01"
    auto_log: true
    keepalive: 60

  - name: bastion
    host: jump.example.com
    port: 22
    user: ops
    identity_file: ~/.ssh/id_rsa
    tags: [prod, bastion]

settings:
  default_user: root
  default_port: 22
  log_dir: ~/.sshman/logs
  connect_timeout: 10
  master_password_salt: <auto-generated>
```

**字段说明：**
- `name`: 唯一标识符，作为操作目标
- `host`, `port`, `user`: SSH 连接参数
- `password`: 明文存储在加密文件中；留空则优先使用 identity_file，其次交互式提示
- `identity_file`: SSH 私钥路径
- `tags`: 标签列表，支持过滤和分组
- `jumphost`: 跳板机 session 名称（链式引用）
- `tunnels`: 端口转发列表（连接时自动建立）
- `notes`: 自由文本备注
- `auto_log`: 是否启用操作日志录制
- `keepalive`: ServerAliveInterval 秒数

## 三、命令行接口

```
sshman connect <name>          # 连接指定会话
  --log / --no-log
  --no-tunnels

sshman list                    # 列出所有会话
  --tag prod,web
  --keyword beijing
  --detail

sshman add                     # 交互式向导添加
sshman add --file config.yaml  # 批量导入

sshman edit <name>             # 编辑会话配置
sshman remove <name>           # 删除会话（需确认）
sshman clone <name> --as new   # 复制并更名

sshman batch <command>         # 批量执行命令
  --tag prod
  --names web,db
  --parallel 5

sshman import                  # 从外部导入配置
  --source ssh-config
  --source ansible /path
  --source csv servers.csv
  --dry-run

sshman check                   # 健康检查（端口连通性）
  --tag prod
  --timeout 3

sshman log [name]              # 查看操作日志
  --last 10
  --date 2026-06-12
  --search "error"

sshman tunnel <name>           # 仅建立端口转发，不进入 shell
sshman init                    # 初始化配置（生成密钥、设置主密码）
sshman encrypt / decrypt       # 手动加密/解密配置文件
```

## 四、模块架构

```
sshman/
├── cli.py              # Click 命令行入口
├── core/

```

### 模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| `crypto` | AES-256-GCM 加解密；Argon2id 密钥派生；主密码哈希 | `cryptography` |
| `config` | YAML 读写；会话 CRUD；标签索引；完整性校验 | `pyyaml`, `crypto` |
| `connector` | pexpect SSH 会话管理；交互式登录；窗口同步 | `pexpect` |
| `jumphost` | ProxyJump 参数构建；多级跳板链 | — |
| `tunnel` | 端口转发参数构建；隧道验证 | — |
| `logger` | 终端输出录制；日志轮转；搜索 | — |
| `checker` | TCP 连通性检测；延迟统计 | — |
| `importer` | 解析 ~/.ssh/config、Ansible inventory、CSV | — |

### 连接流程

```
CLI
 ├─ 1. crypto.decrypt(config_file, master_password)
 │      → 内存明文配置
 ├─ 2. config.find_session("prod-web-01") → Session 对象
 ├─ 3. [如有 jumphost] build_proxy_command()
 │      → ssh -J user@bastion:22 ...
 ├─ 4. tunnel.build_args() → -L 5432:127.0.0.1:5432
 ├─ 5. connector.spawn(session) → pexpect.spawn("ssh ...")
 │      → 处理: host key → 密码 → MFA → shell 就绪
 └─ 6. [如 auto_log] logger.start_recording()
         → ~/.sshman/logs/prod-web-01/2026-06-12_143000.log
```

### 配置生命周期

```
明文 YAML (内存) ──→ crypto.encrypt() ──→ 密文文件 (~/.sshman/config.enc)
密文文件 ──→ crypto.decrypt() (输入主密码) ──→ 明文 YAML (内存，会话生命周期)
```

明文配置仅在内存中存在，进程退出即释放。

## 五、安全设计

| 层面 | 方案 |
|------|------|
| 加密算法 | AES-256-GCM（认证加密） |
| 密钥派生 | Argon2id（memory=64MB, iterations=3, parallelism=4） |
| 主密码 | 首次 `init` 时设置，不存盘；hash salt 存储于配置头 |
| 内存安全 | 明文配置用 `bytearray` 存储，用后 zero-out；抑制日志打印敏感信息 |
| 文件权限 | 配置文件强制 `chmod 600`；日志目录 `700` |
| SSH 实践 | 优先密钥认证；密码仅 fallback；HostKey 严格校验 |
| 解密时机 | 仅在需要时解密，用完立即丢弃 |

## 六、分阶段实施

### Phase 1 — MVP 核心
- 项目脚手架 (`pyproject.toml`, Click CLI)
- `crypto` 模块：AES-256-GCM + Argon2
- `config` 模块：YAML CRUD
- `connector` 模块：pexpect SSH + 密码/密钥/主机确认
- 命令: `init`, `add`, `list`, `connect`, `remove`, `encrypt`, `decrypt`

### Phase 2 — 进阶连接
- `jumphost` 模块：ProxyJump
- `tunnel` 模块：-L/-R/-D 转发
- 命令: `edit`, `clone`, `tunnel`

### Phase 3 — 批量与管理
- `batch` 模块：多服务器命令 + 并发
- `checker` 模块：TCP 健康检查
- 命令: `batch`, `check`

### Phase 4 — 导入与日志
- `importer` 模块：ssh_config / Ansible / CSV
- `logger` 模块：终端录制、搜索
- 命令: `import`, `log`

### Phase 5 — 体验增强
- Shell 自动补全（bash/zsh/fish）
- 密码强度校验和过期提醒
- 会话分组/文件夹
- 配置完整性自检
